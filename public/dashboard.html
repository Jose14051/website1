Encriptación de Contraseñas

Las contraseñas de los usuarios están encriptadas antes de almacenarlas en la base de datos. Esto garantiza que incluso si la base de datos es comprometida, las contraseñas no puedan ser fácilmente descifradas por los atacantes.

Validación de Entrada

Los campos de entrada como el nombre de usuario y la contraseña son validados para prevenir ataques de inyección de código y asegurar que los datos ingresados por los usuarios sean seguros.

Verificación de Token JWT

Antes de permitir el acceso a las rutas protegidas, se verifica la validez del token JWT enviado por el cliente. Esto garantiza que solo los usuarios autenticados puedan acceder a recursos protegidos.

Manejo de Errores Detallado

Se han implementado manejadores de errores detallados para varios escenarios, como errores de registro, inicio de sesión, rutas no encontradas y errores internos del servidor. Esto ayuda a proporcionar mensajes de error claros y evitar la exposición de información sensible en caso de fallos.

Configuración de Clave Secreta Aleatoria

La clave secreta utilizada para firmar los tokens JWT se genera aleatoriamente, lo que mejora la seguridad al hacer que sea más difícil para los atacantes adivinar la clave.

Uso de Cookies Seguras

Las cookies se configuran con opciones seguras para prevenir ataques de robo de cookies y mejorar la seguridad de la sesión del usuario.
